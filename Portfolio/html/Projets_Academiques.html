Projet Application promouvant la culture (Rézisten)

Les compétences mises en œuvre

  • Développer une solution informatique pour un client.
  • Optimisation d'applications en fonction de critères spécifiques (temps d’exécution, précision, consommation de ressources)
  • Installer, configurer, mettre à disposition, maintenir des infrastructures réseaux et optimiser le système informatique d’une organisation
  • Concevoir, gérer, administrer et exploiter les données de l’entreprise et mettre à disposition toutes les informations pour un bon pilotage de l’entreprise
  • Satisfaire les besoins des utilisateurs au regard de la chaîne de valeur du client. Piloter un projet informatique avec des méthodes classiques ou agiles.
  • Acquérir, développer et exploiter les aptitudes nécessaires pour travailler efficacement dans une équipe informatique
  • Nombre de personnes sur le projet : 7
  • Durée : 9 semaine de (2 demie-journée) soit 320j/homme
  • Langages abordés : Mise en place de serveur(2), Php, HTML, CSS, JavaScript, PostreSQL
  • Objectif : Réaliser une application promouvant la culture
  • Contenu du projet :

    • Notre application est hébergée sur 2 serveurs, l'un possédant le code et l'autre possédant la base de données. Nos serveurs sont donc communicant et sécurisés. Pour la partie code, notre application est une histoire dont vous êtes le héros, chaque histoire est en lien avec un lieu. vous retrouverez au sein de chaque histoire 2 questions, la première une question complexe vous obligeant à vous rendre sur le lieu, et si vous ne pouvez-vous y aller, vous pouvez passer à une question plus simple sur le lieu avec une réponse trouvable sur internet. A la fin de chaque chapitre, une description du lieu et sa véritable histoire vous sera présentée.

Projet Application de gestion de marché (LeBonStand)

Les compétences mises en œuvre

  • Développer des interfaces utilisateur
  • Faire des essais et évaluer leurs résultats par rapport aux spécifications
  • Comparer des algorithmes pour des problèmes classiques (tris simples, recherche...)
  • Analyser un problème avec méthode (découpage en éléments algorithmiques simples, structures de données...)
  • Mettre en place les outils de gestion de projet
  • Comprendre les besoins du client et de l’utilisateur
  • Identifier les statuts, fonctions et rôles de chaque membre d’une équipe pluridisciplinaire
  • Acquérir les compétences interpersonnelles pour travailler en équipe
  • Nombre de personnes sur le projet : 6
  • Durée : 3 semaines (~700 heures de travail cumulé)
  • Langages abordés : Java, JavaFX
  • Rendu : Application + 3 oraux + dossier de conception + document de conception des interfaces (personas, maquettes, critères ergonomiques sélectionnés et leur mise en œuvre) + Document Graphes analyse de gestion de tâches + optimisation multicritères + cadrage du projet + bilan
  • Objectif : Réaliser une application de gestion d'événements de bout en bout (gestion d'équipe, gestion des risques,...)

    • Ce projet était notre projet de fin d'année, pluridisciplinaire et en groupe, semblable à une entreprise. Nous avons commencé par la gestion du projet, la direction à prendre, la répartition des tâches, et trouvé une identité de projet. Puis, nous sommes passés à la partie IHM (Interface Homme-Machine) avec la création de la base de notre projet et de nombreux diagrammes afin de bien comprendre le fonctionnement de notre application dans les moindres détails. Ensuite, nous avons commencé la partie développement avec la création des différentes interfaces utilisateur. Nous avons fait des tests et des corrections pour rendre l'application la plus fonctionnelle possible. Enfin, nous avons travaillé sur la communication de notre application avec 3 oraux différents : un oral de marketing, un oral d'anglais et un oral de démonstration de notre application.

Exploitation d’une base de données

Les compétences mises en œuvre

  • Mettre à jour et interroger une base de données relationnelle (en requêtes directes ou à travers une application)
  • Visualiser des données
  • Concevoir une base de données relationnelle à partir d’un cahier des charges
  • Nombre de personnes sur le projet : 2
  • Durée : 2 semaines
  • Langages abordés : PostgreSQL
  • Objectif : Filtrer une base de données pour obtenir des données cohérentes
  • Rendu : Base de données nettoyée et transformée + Rapport statistique en anglais
  • Contenu du projet :

    • Le projet consistait à filtrer une base de données pour obtenir des données cohérentes. Il nous était imposé de partir de la base de données des Nutri-Scores des produits, une base de données open source et donc conséquente en données. Pour notre projet, nous devions filtrer les données sur les volailles aux États-Unis, les rendre cohérentes en enlevant toutes les valeurs aberrantes que l'on pouvait trouver, et cela sous forme de script. Enfin, nous avons rédigé un rapport statistique en anglais sur R pour analyser le sujet des volailles aux États-Unis d'après le Nutri-Score et les informations de la base de données.

Projet création de site web (Hardis groupe)

Les compétences mises en œuvre

  • Appréhender les besoins du client et de l’utilisateur
  • Mettre en place les outils de gestion de projet (Whimsical)
  • Conduire un projet
  • Identifier les statuts, les fonctions et les rôles de chaque membre d’une équipe pluridisciplinaire
  • Acquérir les compétences interpersonnelles pour travailler en équipe
  • Nombre de personnes sur le projet : 3
  • Durée : 3 semaines (1 semaine recueil des besoins + 1 semaine grille de projet et préparation orale + 1 semaine code du site)
  • Langages abordés : HTML, CSS (+ un peu de JavaScript)
  • Objectif : Création d'un site web pour une entreprise du bassin grenoblois
  • Contenu du projet :

    • C'était notre premier projet de l'année et j'ai pris la tête de celui-ci.
    Nous avons commencé par le recueil des besoins de l'entreprise en regardant leur site actuel.
    Nous avons ensuite rempli une grille nous permettant de voir quels étaient les aspects sur lesquels nous avions besoin de progresser. Mais aussi une grille concernant les changements dont nous avions besoin pour rendre notre site le plus proche possible de ce que voulait l'entreprise (la consigne étant de leur refaire un site web facilitant l'insertion des élèves de troisième pour leur stage).
    Enfin, nous avons codé le site en utilisant les outils que nous avions vus en cours, notamment Visual Studio Code pour la partie HTML, CSS, et JavaScript.
